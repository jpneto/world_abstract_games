> I'm interested in mutators, and get the idea that "mutator" is just
> another name for "game rule". Is this the case? In chess variants,
> mutators can change any variant into any other, so is there really a
> difference between a mutator and a rule [other than the spelling and
> pronunciation]?

Yes, mutators is basically a game rule, but with enough flexibility
that it can be used, a bit like a function, to transform a game into
another game. Most (all?) mutators are not universal functions
(meaning, they don't work with all games) but some are very flexible.
One of those is the PIE rule.

The main idea of mutators is to provide the game designer with a set
of tools to define or refine his game project.

Chess variants was the first game set I considered, but they also work
in other abgames. In Sagme's blog you can find dozens of games that
appeared basically by mutator manipulation.

>My original question was: "can gameboards be mutators?"

you mean, like the 8x8 board? I tend to separate the hardware (board,
pieces...) from the software (rules/mutators). Sure, a certain
material can provide inspiration for new rules, but usually I try to
think of an idea first and get the hardware as needed.
