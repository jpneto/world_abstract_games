I strongly advise the owner to do so, so we can share our pretty ascii boards :-) About CGT, a game must have the following properties: a) Two players in alternate single-move turns b) Perfect Information (no random devices, no hidden info) c) All branches of the game tree are finite (ie, no matter the move sequence, even if the game had a pass option, the game always ends). d) The last player to move wins the game (there's also a misÃ¨re version, where the last mover loses) Here is a nice CGT game which no one was able to analyse (so far), called WInerack: WINERACK 20 =========== The setup is a row of 20 bottles. A bottle is available if it doesn't support any other bottle. A move is picking an available bottle and place it over two adjacent bottles at the same level. However, a moved bottle must always move to a higher position. Last one to move, wins. We (=Richard Hoshino, Paul Ottaway and myself) looked at Winerack when played as a partizan game using red wine and white wine bottles, in the start position they alternate. Left moves the white wine and Right the red wine bottles. We haven't been able to answer the basic question, a puzzle really: given n white wine bottles in a row, how many moves does Left have? (He wants the maximum number of moves.) We conjecture that using a greedy algorithm works (moving a bottle the least number of levels) but that can lead to a situation where a bottle is lifted to many levels whereas this can be avoided by being less than greedy at an earlier stage.
